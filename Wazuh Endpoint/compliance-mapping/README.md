# GRC Compliance Readiness Lab Using Wazuh

## Overview
This project demonstrates how Wazuh can support Governance, Risk, and Compliance readiness for a small healthtech startup.

The goal is to show how technical security evidence from Wazuh can be mapped to compliance frameworks such as SOC 2, HIPAA, ISO 27001, NIST, GDPR, and HITRUST-style requirements.

## Scenario
HopeHealth AI is a fictional healthtech startup that stores sensitive patient intake information. The company wants to prepare for customer security reviews, SOC 2 readiness, HIPAA readiness, and future compliance audits.

## Tool Used
- Wazuh SIEM/XDR

## Wazuh Capabilities Used
- File Integrity Monitoring
- Vulnerability Detection
- Security Configuration Assessment
- Authentication Monitoring
- Security Alerts
- Incident Response Evidence

## Frameworks Covered
- SOC 2
- HIPAA
- ISO 27001
- NIST
  

## What This Project Shows
- How to collect compliance evidence from Wazuh
- How to map security alerts to compliance controls
- How to identify risks and gaps
- How to create a remediation plan
- How to prepare an auditor-style summary

## Key Skill Demonstrated
This lab shows the ability to translate technical security findings into compliance, risk, and audit language.
