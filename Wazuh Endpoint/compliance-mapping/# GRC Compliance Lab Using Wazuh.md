# GRC Compliance Readiness Lab Using Wazuh

## Project Purpose

This lab demonstrates how Wazuh can be used as a technical evidence collection tool for Governance, Risk, and Compliance (GRC) work.

The goal is not just to show that Wazuh generates alerts, but to show how technical security evidence can be translated into compliance, audit, risk, and remediation language.

This lab uses:

- Ubuntu as the Wazuh Manager
- macOS as the Wazuh Agent endpoint
- Wazuh Dashboard for reviewing alerts and evidence
- GitHub for documentation

## Role Alignment

This lab supports skills needed for an IT Security and Compliance Analyst role:

- SOC 2 readiness
- HIPAA readiness
- ISO 27001 concepts
- NIST control concepts
- GDPR security concepts
- HITRUST-style readiness
- Risk assessment and gap analysis
- Remediation tracking
- Audit evidence collection

## Big Idea

A technical analyst might say: "Wazuh generated an alert."

A GRC analyst should say: "Wazuh generated evidence that supports a security control. I reviewed the alert, identified the risk, documented the gap, and recommended remediation."

That translation is the core skill this lab demonstrates.
